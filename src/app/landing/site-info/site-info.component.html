About this site

I designed this site for demo purposes as an Angular SPA (Single Page Application) hosted on Firebase.

Here are just a few topics I want to specifically touch on for now:

    Tech Overview

      Here is a list of most of the concepts used in the development of this site:

    • Angular Components
    • Data Binding
    • String Interpolation
    • Event Binding
    • Directives (*ngFor, *ngIf, *ngClass etc.)
    • Routing
    • Template Driven Forms
    • Observables/Rxjs basics
    • Firebase Authentication
    • Feature Modules
    • Lazy Loaded Front End Routes
    • Module Pre Loading
    • Debugging
    • Lots of Debugging

    • Services (primarily the Auth service, but
      there is also a small "App" service that handles
      a few minor tasks app-wide)
    • HTTP Post Requests
    • Handling HTTP Response Data
    • HTTP Error Handling

    • Backend:
    • Firebase Auth
    • Firebase Firestore Database
    • Firebase Trigger Email Extension for the contact forms

    • Hosting:
    • Firebase Hosting

  • Home

    Just below the welcome hero lies the Auth component which provides the user interface for the Sign Up and Sign In functionality.

    In the UI, once the user signs up or signs in:

    • The Auth component is removed from the DOM.
    • The Logout button is rendered in the header.
    • The user's login email is shown in both the header (collapsed only) and footer.
    • The user's name appears in the Welcome hero.
    • When the user clicks on the Blog link they are no longer redirected to Auth, the Blog route is rendered.

  • Authorization

    Sign Up Sign Up, Log InLog In and Log Out functionality is handled programmatically in the front end code and supported by Firebase Authentication on the back end.
