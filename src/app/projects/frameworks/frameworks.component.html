Inventory

"Inventory" is an MVP inventory management app for the foodervice industry.

Development began about 25 May 2022, it is still in it's very early stages.

As of 24 June 2022:

Users can create Owner, Manager and Staff accounts with different permissions for each within their organization.

  • Owners can:
    • Define their business organization within the app
    • Define each of their business locations
    • Assign access permissions for Managers and Junior Staff
    • Or, unrestricted, just use the app themselves as they like
  • Managers, for each location they are assigned to manage can:
    • Create new products
    • Create or approve current working inventory
    • View and filter historical inventory by date
  • Junior Staff can enter inventory counts for each location they are assigned and submit them for approval by a Manager or Owner.
  • Image Uploads:
    • User avatars
    • Business avatars
    • Coming soon: Location avatars, Product photos
  • Some of the planned improvements:
    • Landing page content.
    • Massive UX/UI improvements.
    • More reusable app components. Much is redundant.
    • Change email/password feature.
    • Owner/Manager Dashboards.
    • Instant messaging feature.
    • App Notifications.
    • Known bug fix: after login, the first API call fails authentication, subsequent requests all succeed.
    • Flutter mobile companion app.

Boggle +

Boggle+ is my very first MEAN stack application!
It uses every concept I've learned in Angular so far plus all of the other elements of the MEAN stack.

The front end is a demonstration of Angular Material design, reactive forms and the majority of the fundamental features of the Angular framework.

The back end uses Node.js/Express.js in a split approach with a "front end" server that communicates to a "back end" server that utilizes a REST API to handle requests, responses, errors and communicates with a MongoDB database to create, read, update and delete data.

Here is a list of some key Angular features used in Boggle+

  • Component Modules and Routing
  • Data Binding
  • Event Binding
  • Directives
  • Angular Services
  • Angular Router
  • Reactive Forms
  • File uploads with Multer
  • Observables/Rxjs basics
  • Simple authentication logic using JSON Web Token and Bcrypt js
  • Lazy Loaded Routes
  • Pre Loading
  • Creating a Node.js Server with Express.js
  • HTTP Requests
  • Handling HTTP Response Data
  • REST APIs
  • Backend Routes, Controllers
  • HTTP Error Handling
  • MongoDB Integration
  • AWS EC2/S3 Deployment

Shefbook

Shefbook is a recipe book app that incorporates much of what you might see in a classic "to-do list" style app.

In Shefbook, users can:

  • Sign Up, Log In and Log Out handled programmaticaly in an authorization component, and Firebase Authorization
  • Fetch existing recipes from a Firebase Realtime Database
  • Create new recipes for the community
  • Edit recipes
  • Send recipe ingredients lists to their locally stored shopping list
  • Add, edit, delete shopping list items
